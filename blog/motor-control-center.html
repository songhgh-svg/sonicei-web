MCC Commissioning: What the Factory Test Doesn't Tell You

A motor control centre that passed its factory acceptance test can still fail catastrophically on site. The FAT tests the equipment in a controlled environment. It does not test the installation. Here's what changes between the factory floor and the field — and how to catch it before energisation.

The FAT certificate is not a commissioning certificate. This distinction matters, and it gets blurred on more projects than it should. When a project team receives a fully tested MCC with a comprehensive factory test report, there is a natural tendency to treat the on-site commissioning as a formality — a visual inspection, a few checks, and then energise.

The MCC that arrives on site is the same hardware that was tested. The installation connecting it to the rest of the system is not. Every field termination, every conduit entry, every cable gland, every outgoing cable, every interlock wiring connection — none of that existed during the FAT. All of it can introduce defects.

Transit and Installation Damage

The first category of defects that FAT-tested equipment develops is physical. Marine transport to offshore locations and road transport to remote onshore sites subjects MCC enclosures to vibration, shock loads, and humidity exposure that can loosen terminations, unseat draw-out module contacts, and introduce moisture into sealed compartments.

The standard site receipt inspection — a visual check confirming the MCC arrived without obvious mechanical damage — does not catch these defects. A thorough site receipt inspection for an MCC includes opening every draw-out module, checking contact wear, verifying that all busbars are fully seated and that no terminations have vibrated loose in the main incomer section. This takes time. It is worth the time.

"The MCC had a perfect FAT report. When we opened the incomer section on site, three main busbar bolts were finger-tight. The vibration during sea freight had backed them out. If we'd energised without checking, we'd have had an arc flash event within the first week of operation."

Field Wiring: The New Failure Surface

Once the MCC is physically verified, the focus shifts to field wiring — and this is where the volume of potential defects expands significantly. A medium-sized MCC serving 40 motor starters will have hundreds of individual field wiring terminations in the control section alone, plus power cable terminations at every outgoing feeder.

Each termination is a potential failure point. The failure modes are well known: incorrect cable identification leading to wrong-feeder termination, incorrect phase rotation at motor terminals, missing or wrong-value overload relay settings, control wiring polarity reversal causing contactors to fail-energise, and star-delta transition timer settings not matching motor specifications.

23% of MCC outgoing feeders have at least one wiring defect on first check

Overload Relay Setting: The Most Commonly Wrong Parameter

Thermal overload relay settings are the single most frequently incorrect parameter found during MCC commissioning audits. The setting should correspond to the motor's rated full-load current, with appropriate derating for ambient temperature and starting duty. In practice, overloads are frequently found set to their maximum value — effectively bypassing the protection — or set to the same value across all feeders regardless of the motor ratings they serve.

This isn't carelessness. It's the consequence of a process gap: overload settings are defined in the motor schedule in the electrical design package, but there is often no formal step in the installation workflow that transfers those settings to the field. The electrician installs the overload relay, sets it somewhere reasonable-looking, and moves on.

// MCC Site Commissioning — Critical Checks

Phase rotation verification at every outgoing motor terminal — before first start, no exceptions. Overload relay setting: confirmed against motor FLA from nameplate and motor schedule. Control wiring functional test: local start/stop from MCC, remote start/stop from DCS, feedback signals verified. Interlock simulation: all process interlocks exercised with motor isolated. Star-delta timer: confirmed against manufacturer recommendation for motor kW rating.

The Integration Test Gap

The final category of MCC commissioning failure is the integration test — verifying that the MCC communicates correctly with the DCS or PLC that controls it. FAT typically tests the MCC in isolation or with a simulation panel representing the control system. It does not test the actual DCS I/O, the actual communication network, or the actual control logic as programmed for this project.

Integration faults — wrong DCS address mapping, inverted feedback signals, missing permissive logic — are consistently discovered only at the full system test stage, when the cost of correction has multiplied relative to what it would have been if caught during dedicated MCC-to-DCS loop checks.

Allocating time for a systematic, feeder-by-feeder MCC integration check against the control narrative — before the system-level functional test — is the investment that prevents the integration test from becoming an extended fault-finding exercise.
